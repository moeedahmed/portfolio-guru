# Hero Demo Case — Synthetic CBD / Mini-CEX

Date: 2026-06-17
Owner: Founder / Portfolio Guru
Status: Synthetic asset for the Hermes hackathon rehearsal and demo. Not a
real patient encounter. No real names, no real identifiers, no invented
personal facts.

## Why one hero case

The hackathon demo only needs one clean case to walk the full Portfolio
Guru golden path:

> shift note → structured RCEM Kaizen draft → Portfolio Health gap →
> draft preview → human approval → Kaizen draft save → dashboard record
> → earn/spend/ledger view.

A single hero case keeps the recording short, keeps the rehearsal
deterministic, and removes the "which case do we use this take?"
question on demo day.

## Persona

- Persona name: `Demo Trainee`. Used only inside the bot session for
  display. Not a real person.
- Training level: `ACCS CT2` (Royal College of Emergency Medicine 2025
  curriculum).
- Why ACCS: the 2025 curriculum is the most common Kaizen path in the
  Portfolio Guru beta cohort, and a CBD-on-chest-pain reflection is a
  realistic ACCS-stage encounter that does not require advanced
  registrar-only capabilities.

## Source shift note

Verbatim text used as the Telegram message in the rehearsal. It is the
kind of voice-to-text dump a CT2 might send after a busy resus shift.
Identifiers are stripped: no name, no date of birth, no NHS number, no
postcode, no admission number, no specific hospital, no specific
consultant. Clinical detail is generic to the presentation.

```
Late shift in resus. Middle-aged adult brought in by ambulance with
central crushing chest pain, ongoing for about an hour. Sweaty, looked
unwell. ECG showed dynamic ST changes in the inferior leads. Troponin
came back elevated on the second draw. Treated as ACS — aspirin,
ticagrelor, fondaparinux per local pathway, opioid analgesia, GTN
infusion. Discussed with cardiology on call, accepted for primary PCI.
Reflecting: I should have repeated the 12-lead earlier when the pain
pattern shifted — would have triggered the cath lab activation a bit
faster. Good handover to the cardiology team. No adverse outcome.
```

The text is intentionally messy in the way real shift notes are messy
(no patient demographics beyond age band, no investigations in fixed
order, reflection bolted on at the end).

## Recommended form path

- Primary recommendation: `CBD` (Case-Based Discussion) — reflective
  discussion of an acute coronary case, exactly the shape RCEM Kaizen
  expects for an ACCS CT2 encounter at SLO1 / SLO3.
- Secondary recommendation: `MINI_CEX` — observed clerking with focused
  exam. Acceptable alternative if a consultant directly observed the
  clerking; for this case we assume the encounter is reflective only,
  so CBD is the demo path.

## Portfolio Health gap surfaced

For the demo, the natural gap to highlight is:

- `Learning need not recorded` — the case does not include an explicit
  learning need, so Portfolio Health flags it as a directional gap
  before ARCP. Phrased as "planning aid, not official outcome".

This keeps Portfolio Health visible without inventing data and stays
consistent with the "directional, not official" framing required in
`docs/PORTFOLIO_HEALTH_SPEC.md`.

## Safety properties of this asset

- No real names. The only person string used in the bot session is the
  static `Demo Trainee` label.
- No patient identifiers. Age is given as an age band, presentation is
  generic, no hospital/admission/identifier-like strings appear.
- No invented credentials. Kaizen sign-in is performed manually by the
  founder during rehearsal; this asset never contains a username or
  password.
- No invented assessment outcome. The draft is preview-only; the demo
  never claims the case has been signed off, approved, or accepted by a
  supervisor or RCEM.
- No claim of RCEM endorsement. The asset is independent demo content.

## How this asset is used

1. The text block under "Source shift note" is pasted (or read aloud
   via voice) into the bot during rehearsal/recording.
2. The structured draft table above is the expected preview state.
3. The Portfolio Health gap above is the expected ARCP-health
   surfacing.
4. The Kaizen draft save step uses founder-supplied credentials and a
   human-approved tap. The asset itself never contains credentials.
5. If live Kaizen is slow or blocked, the rehearsal falls back to the
   deterministic mocked path documented in
   [`REHEARSAL_RUNBOOK.md`](REHEARSAL_RUNBOOK.md) — no patient data is
   invented to fill the gap.
